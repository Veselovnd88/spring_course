Скоуп задает принцип создания бинов
скоуп сингл тон - по умлочанию - создается только один объект,
при вызове метода гетБин возвращается ссылка на один и тот же объект
firstMusicPlaye = getBean
secondMusicPlayer = getBean -это будет тот же самый объект
этот скоуп рпименяется когда у бина нет изменяемых состояний (stateless)
иначе - если меняется состояние - то оно меняется у всех пользователей(например изменяется какое то поле)
можно скоуп указатб явно "scope = "singleton"
следующий скоуп - называется prototype
каждый раз создает новый объект - разные объекты
Чаще всего используется когда бины stateful - когда состояние меняется, для нашего случая
с плеером лучше сделать прототип
дугие скоупы - сесшн, глобал сешн, реквест
